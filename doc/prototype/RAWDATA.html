Inventory Emissions Data Files

Raw data source locations may be specified either in AIRS "UTM" (with its nonstandard kilometer units) or in decimal Lat-Lon coordinates. Coordinates are interpreted as being Lat-Lon when the UTM Zone field (columns 100-101 in PRAW) in the record either is blank or is negative (e.g., the "-9" conventionally used for missing in AIRS-related data).

ARAW: AMS ASCII area source emissions inventory file
PRAW: AFS ASCII point source emissions inventory file
PRDAY and PRHOUR: AFS ASCII point source period-specific (day-specific or hour-specific) emissions inventory file

Profile/Factor/Coefficient Data

Generally, profile files consist of lines with a profile-index field and some set of fields for factors. These files may be cut down to contain only records for actually-used profiles, in the interest of faster execution and improved resource usage. Generally, they should be sorted on profile index (using the UNIX sort utility), to prepare for the binary lookups in the programs which comsume these files. A special case are the temporal profiles files ATPRO and PTPRO, which contains multiple "packets"--subfiles, actually--each with its own index set, for the monthly, weekly, and diurnal temporal allocation factors. Library routine RDTPROF() sorts each of these subfiles on the fly, so that ATPRO, MTPRO, and PTPRO need not be sorted manually by the user during data preparation phase. Non-AIRS profile files are the mobile-source related MVMTM VMT mix profiles file and the various MOBILE-5/EMISFAC-related inspection and maintenance profiles for mobile source emissions.

Cross-Reference Data

Generally, cross-reference files are AIRS AFS cross reference files cut down by the relevant programs (GRDAMAT, SPCAMAT, SPCMMAT, SPCPMAT, TMPAREA, TMPMOBIL, and TMPPOINT) from the files AIRS exports to apply only to the set of sources which actually occur in the region being studied, and then are sorted by source (e.g., by FIPS code and ASC code for area source cross reference files). This has the effect of substantially reducing the memory requirements and CPU consumption of the subsequent processing.

Special cases of non-AIRS cross reference files are the ZONES time zone cross reference file, described further in a later section, and the PSTK missing-stack-parameter cross reference file used by programs ELEVPOINT and LAYPOINT. Other special cases are the MPLIST mobile source emissions factor cross reference file and the MPREF emission factor parameters cross reference file.

The list of cross-reference files is the following:

AGREF: Area source gridding cross-reference file
MGREF: Mobile source gridding cross-reference file
ASREF: Area source speciation cross-reference file
MSREF: ASCII mobile source speciation cross-reference file
PSREF: Point source speciation cross-reference file
ATREF: Area source temporal cross-reference file
MTREF: ASCII mobile source temporal cross-reference file
PTREF: Point source temporal cross-reference file
MPLIST: mobile source emissions simulation-specific x-ref list (indexes into MEFS)
MPREF: emission factor parameters x-ref file
PSTK: Point source stack parameter cross-reference file
ZONES: FIPS code to time zone cross reference file

Grid and Time Zone Data

Because SMOKE is intended to drive air quality models at a variety of scales from urban through regional, and to support multiple grids (e.g., in nested-grid situations) at the same time, it requires a more sophisticated way of dealing with spatial issues than does the UAM's EPS. The GRIDDESC file provides a single master site for the storage of all information about all of the grids and coordinate systems (map projections) to be used in a study.

Because of the potentially regional scale of EDSS/Models-3 simulations, special care needs to be taken with time zone information, as well (since a regional simulation may span multiple time zones). The ZONES time zone cross reference file provides a means of dealing with this problem. Note that some regions (e.g., regions containing Indianapolis, which does not implement Daylight Savings Time, as well as non-Indiana areas which do) will require two time-zone files to deal with episodes crossing over between Standard Time and Daylight Savings Time

GRIDDESC: EDSS/Models-3 grid and coordinate system names and specifications file
ZONES: FIPS code to time zone cross reference file

Auxiliary Lists Constructed from Raw Data

The existing prototype is dimensioned exactly by array bounds determined by the input emissions inventory. The point source emissions vectors, for example, are dimensioned by the exact number of point sources found in the inventory. These dimensions can be found by applying the UNIX utilities cut, sort -u, and wc -l to the inventory, as in the following script: 
#
#   Construct actual point sources (FIP+ASC+PLANT+STACK) list
#   setenvs for input and output files
#
setenv PRAW     <physical path name for AIRS AFS point source file
setenv PTSRCS   <physical path name for output file
#
#
#
#  FIPS  is cols 12-16
#  SCC   is cols 29-38
#  PLANT is cols 40-44
#  STACK is cols 46-48
#
#  blanks occur between fields in PRAW
#
#  note SCC has alignment problems:  8 digits in 10-space field,
#  necessitating "sort -n"
#
cut -c 12-17,29-48 < $PRAW | sort -n -u > $PTSRCS
#
#   done
#
The actually-occurring source category files ASCC and PSCC are also used by various programs for excluding cross reference records not applicable to the study they are processing.

The auxiliary list files are:

AFIP: Actually occurring area source FIPs
PFIP: Actually occurring point source FIPs
ASCC: Actually occurring area source ASCs
PSCC: Actually occurring point source SCCs
ASRCS: Actually occurring area sources (FIP+ASC)
PSRCS: Actually occurring point sources (FIP+SCC+PLANT+STACK)
POLCODES: Actually occurring inventory species codes
