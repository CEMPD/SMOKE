ARDIMS3.EXT: Area-Source Dimensions

In the current prototypes, area source naming and dimensioning information are kept in the Fortran INCLUDE-file ARDIMS3.EXT. This file is used by various routines in the following programs:
  • CSGAREA
  • CSGMERGE
  • CSGLMERGE
  • CTLAMAT
  • GRDAMAT
  • GRDAREA
  • RAWAREA
  • SPCAMAT
  • TMPAREA
  • FIPXY2CR

Parameters specified in this file are the following

  • NAFIP, the number of area-source FIPS codes appearing;
  • NASID, the number of area-source state codes appearing;
  • NASCC, the number of area-source SCC codes actually occurring in the inventory;
  • NASRG, the number of surrogate categories actually occurring in the inventory;
  • NSPRA, the number of area-source speciation profiles actually occurring in the inventory;
  • NACEL, the maximum number of area sources per cell;
  • NCFIP, the maximum number of cells per FIP (i.e., per county);
  • NTRFA, the maximum number of FIP-ASC pairs actually occurring in the temporal cross-reference file; and
  • NASRC, the number of area-sources actually occurring in the inventory.

Source code for the North Carolina UAM domain version of ARDIMS3.EXT

PTDIMS3.EXT: Point-Source Dimensions

In the current prototypes, point source naming and dimensioning information are kept in the Fortran INCLUDE-file PTDIMS3.EXT. This file is used by various routines in the following programs:
  • CSELEV
  • CSGPOINT
  • CSGLPOINT
  • CSGMERGE
  • CSGLMERGE
  • CTLPMAT
  • PDPOINT
  • ELEVPOINT
  • GRDPMAT
  • GRDPOINT
  • RAWPOINT
  • SPCPMAT
  • SRTPTX
  • TMPPOINT

Parameters specified in this file are the following

  • NPFIP, the number of point-source FIPS codes appearing;
  • NPSID, the number of point-source state codes appearing;
  • NPSCC, the number of point-source SCC codes actually occurring in the inventory;
  • NSPRP, the number of point-source speciation profiles actually occurring in the inventory;
  • NPCEL, the maximum number of point sources per cell;
  • NSREF, the maximum number of entries in the point-source speciation cross reference file;
  • NTRFP, the maximum number of FIP-ASC pairs actually occurring in the point-source temporal cross-reference file; and
  • NPSRC, the number of point-sources actually occurring in the inventory.
  • NPDAY, the maximum number of point-sources for which there exists day-specific data.
  • NPHR, the maximum number of point-sources for which there exists hour-specific data.
  • NPELV, the maximum number of point-sources to be given elevated source plume-rise treatment.
  • MAXSCN, the maximum number of emissions factor scenarios in input file
  • MAXTMP, the maximum number of emissions factor input temperatures
  • NLIST, the maximum number of emissions factor scenario specific parameter list entries
  • NPREF, the maximum number of emissions factor normal parameter cross reference entries
  • NNFAC, the number of normal cross reference pointers
  • NCFAC, the number of control cross refererence pointers

Source code for the North Carolina UAM domain version of PTDIMS3.EXT

MBDIMS3.EXT: Mobile Source Dimensions and lists

In the current prototypes, mobile source naming and dimensioning information are kept in the Fortran INCLUDE-file MBDIMS3.EXT. This file is used by various routines in the following programs:
  • CSGMOBIL
  • CTLMMAT
  • EMISFAC
  • GRDMMAT
  • GRDMOBIL
  • RAWMOBIL
  • SPCMMAT
  • TMPMOBIL

Parameters specified in this file are the following

  • MXEMIS, the maximum number of emissions types
  • MXFACS, the maximum number of entries in any emissions factors table
  • MXSURG, the number of surrogate categories
  • MXLPC, the maximum number of links per FIP and road-class
  • MXLSRC, the maximum number of link sources
  • MXVMTL, the maximum number of entries in the VMT list
  • MXSTA, the maximum number of state codes
  • MXLNK, the maximum number of link sources
  • MXFIP, the maximum number of FIP county codes
  • MXCFIP, the maximum number of cells per FIP
  • MXSCEL, the maximum number of sources per cell
  • MXGCOF, the maximum number of gridding/ungridding coefficients
  • NSPRM, the maximum number of speciation profiles
  • NETYPE, the number of emissions types
  • NMSRC, the number of mobile sources
  • NRCLAS, the number of road classes
  • NVTYPE, the number of vehicle types

Name and ID-code tables found in this INCLUDE file are the following:

  • MROADS3( NRCLAS ), Road class codes
  • RCLAS3 ( NRCLAS ), Road class names (given with matching MROADS codes, as above):
    • 110: RURAL INTERSTATE
    • 130: RURAL PRIN ART
    • 150: RURAL MINOR ART
    • 170: RURAL MAJOR COL
    • 190: RURAL MINOR COL
    • 210: RURAL LOCAL
    • 230: URBAN INTERSTATE
    • 250: URBAN FREEWAY
    • 270: URBAN PRIN ART
    • 290: URBAN MINOR ART
    • 310: URBAN COLLECTOR
    • 330: URBAN LOCAL
  • VTYPE3 ( NVTYPE ), names of vehicle types
  • VHDESC3( NVTYPE ), descriptions of vehicle types
    • LDGV: Light Duty Gas Vehicles
    • LDGT1: Light Duty Gas Trucks 1
    • LDGT2: Light Duty Gas Trucks 2
    • HDGV: Heavy Duty Gas Vehicles
    • LDDV: Light Duty Diesel Vehicles
    • LDDT: Light Duty Diesel Trucks
    • HDDV: Heavy Duty Diesel Vehicles
    • MC: Motorcycles
  • EMTLST ( MXEMIS ), Emissions factors names list (all possible types)
  • EMTUNIT( MXEMIS ), Emissions factors units
  • EMTDESC( MXEMIS ), emissions factors descriptions. These are given below, together with the matching EMTLIST and EMTUNIT names:
    • EXH_CO (g/mi): EFs for Exhaust CO
    • EXH_NOX (g/mi): EFs for Exhaust NOX
    • EXH_VOC (g/mi): EFs for Exhaust VOC
    • EVP_VOC (g/mi): EFs for Evaporative VOC
    • RNL_VOC (g/mi): EFs for Running VOC
    • RST_VOC (g/mi): EFs for Resting VOC
    • RF1_VOC (g/mi): EFs for Refueling 1 VOC
    • RF2_VOC (g/mi): EFs for Refueling 2 VOC
    • HOT_VOC (g/mi): EFs for Hot soak evaporative VOC
    • CRC_VOC (g/mi): EFs for Crankcase VOC
    • RHR_VOC (g/hr): EFs for Resting by the hour VOC
  • EMTNAM ( NETYPE ) Emissions types as actually used for the SMOKE scenarios: (EXH_CO, EXH_NOX, EXH_VOC, EVP_VOC, RNL_VOC, RST_VOC)
Source code for the North Carolina UAM domain version of MBDIMS3.EXT

  • RAWBIO
  • GRDBIO
  • TMPBIO

Parameters specified in this file are the following

  • TREETY, the number of tree types (pine, decid, conif)

  • FORE, the LUSE subscript for forest
  • WETL, the LUSE subscript for wetlands
  • AGRI, the LUSE subscript for agriculture
  • RMTREE, dimension for OTHERID tree table
  • SPTREE, dimension for SPFORID tree table
  • FACTEN, conversion factor, M^2/hectare
  • BSPCS, the number of biogenic emissions species
  • MSPCS, the number of model emissions species
  • ISOP, the biogenic emissions species subscript for isoprene
  • MONO, the biogenic emissions species subscript for monoterpenes
  • OVOC, the biogenic emissions species subscript for other VOCs
  • NO, the biogenic emissions species subscript for NO
  • NNO, the output model emissions species subscript for NO
  • NPAR, the output model emissions species subscript for parrafins
  • NOLE, the output model emissions species subscript for olefins
  • NALD, the output model emissions species subscript for aldehydes
  • NISO, the output model emissions species subscript for isoprene
  • DENISO, the inverse molecular weight for isoprene
  • DENMON, the inverse molecular weight for monoterpenes
  • DENUNK, the inverse molecular weight for other VOCs
  • DENNO, the inverse molecular weight for NO
  • MON2PAR, the conversion factor of MON to PAR
  • MON2OLE, the conversion factor of MON to OLE
  • MON2ALD2, the conversion factor of MON to ALD2
  • OTH2OLE, the conversion factor of OTH to OLE
  • OTH2PAR, the conversion factor of OTH to PAR

Name-tables found in this INCLUDE file include the following

  • BIOLUSE( LUSES ), biogenic land use categories

    • FORE: forested
    • WETL: wetlands
    • AGRI: agricultural
  • BIOLTYPE( BTYPES )
    • PINE: pine forest
    • DECD: deciduous forest
    • CONF: other coniferous
    • AGRC: agricultural
    • LEAF: leaf area
    • OTHR: other biogenic source
  • BIOSPC( BSPCS ), biogenic-model species
    • ISOP: isoprene
    • MONO: monoterpenes
    • OVOC: other VOCs
    • NO
  • EMSPC( MSPCS ), emitted model species, which for the Carbon-Bond IV prototye are the following:
    • NO
    • PAR
    • OLE
    • ALD2
    • ISOP
  • OTHERID( RMTREE ), tree-types list
    • Mixf
    • Conf
    • Harf
    • Borf
    • Spin
    • Ufor
    • Oak
    • Odcd
    • Cnif
  • SPFORID( SPTREE )

    • Wcnf

    • Wmxf
    • Wwdl

CHDIMS3.EXT: Chemical Species and Mechanisms

In the current prototypes, chemistry naming and dimensioning information for both the inventory species and the model-mechanism species are kept in the Fortran INCLUDE-file CHDIMS3.EXT. This file is used by various routines in the following programs:
  • CSGAREA
  • CSGPOINT
  • CSGLPOINT
  • CSGMERGE
  • CSGLMERGE
  • CTLAMAT
  • CTLPMAT
  • CSGPOINT
  • PDPOINT
  • GRDAMAT
  • GRDPMAT
  • GRDAREA
  • GRDPOINT
  • RAWAREA
  • RAWPOINT
  • SPCAMAT
  • SPCPMAT
  • TMPAREA
  • TMPPOINT

Parameters specified in this file are the following

  • NIPOL, the number of inventory pollutants; and
  • NMPOL, the number of model-mechanism pollutants.
  • COFAC, the inverse of the molecular weight for model species CO.
  • NOFAC, the product of the inverse of the molecular weight for model species NO and the NOx_to_NO conversion factor.
  • NO2FAC, the product of the inverse of the molecular weight for model species NO2 and the NOxNO2 conversion factor.

Data arrays contained in this INCLUDE-file are the following

  • INTEGER PCODE( NIPOL ) containing the sorted list of numeric pollutant codes for the inventory pollutant species. For the North Carolina UAM-domain Carbon-Bond IV emissions prototype, these are:
    • 42101 for CO (carbon monoxide);
    • 42603 for NOX (oxides of nitrogen); and
    • 43104 for VOC (volatile organic compounds).
  • CHARACTER*16 ACODE( NIPOL ) containing the (left-justified) five-digit character-string representations for the numeric inventory pollutant species codes;
  • CHARACTER*16 EINAM( NIPOL ) containing the ordinary chemical names (CO, NOX, VOC) for the inventory pollutant species (in the same order as they are found in arrays PCODE and ACODE (and also left-justified); and
  • CHARACTER*16 EMNAM( NMPOL ) containing the (left-justified) character-string chemical names for the model-mechanism pollutant species. For the North Carolina UAM-domain Carbon-Bond IV emissions prototype, these are:
    • CO, carbon monoxide;
    • NO, nitric oxide;
    • NO2, nitrogen dioxide;
    • OLE, olefins;
    • PAR, parafins;
    • TOL, toluene;
    • XYL, xylene;
    • FORM, formaldehyde;
    • ALD2, high molecular weight aldehydes;
    • ETH, ethene;
    • MEOH, methanol;
    • ETOH, ethanol; and
    • ISOP, isoprene
  • CHARACTER*16 SFNAM( NMPOL ) containing the (left-justified) character-string names for the variables in speciation-factors matrices. The corresponding variables are the factors for converting tons of the inventory species to moles of the corresponding model species. For the North Carolina UAM-domain Carbon-Bond IV emissions prototype, these are:
    • CO_to_CO , for carbon monoxide;
    • NOX_to_NO, for nitric oxide;
    • NOX_to_NO2, for nitrogen dioxide;
    • VOC_to_OLE, for olefins;
    • VOC_to_PAR, for parafins;
    • VOC_to_TOL, for toluene;
    • VOC_to_XYL, for xylene;
    • VOC_to_FORM, for formaldehyde;
    • VOC_to_ALD2, for high molecular weight aldehydes;
    • VOC_to_ETH, for ethene;
    • VOC_to_MEOH, for methanol;
    • VOC_to_ETOH, for ethanol; and
    • VOC_to_ISOP, for isoprene
  • REAL EMWT( NMPOL ) containing the molecular weights for the model-mechanism pollutant species.

Source code for the UAM-Carbon Bond IV version of CHDIMS3.EXT

TMDIMS3.EXT: Temporal Allocation Parameters

In the current prototypes, temporal allocation dimensioning information and related constants are kept in the Fortran INCLUDE-file TMDIMS3.EXT. This file is used by various routines in the following programs:
  • CSGPOINT
  • PDPOINT
  • RAWAREA
  • RAWPOINT
  • TMPAREA
  • TMPPOINT

Parameters specified in this file are the following

  • NMCOD, the maximum number of monthly temporal allocation profile codes in the temporal profile file;
  • NWCOD, the maximum number of weekly temporal allocation profile codes in the temporal profile file;
  • NDCOD, the maximum number of diurnal temporal allocation profile codes in the temporal profile file;
  • NTCOD, the maximum temporal profile table size,
    NMCOD * NWCOD * NDCOD
  • MXSRC, the maximum number of sources (any type)
  • SEC2DAY, the number of seconds in one day, 86400
  • SEC2HR, the number of seconds in one hour, 3600
  • DAY2YR, the number of days in one year, taken as 365.0, for converting daily emissions estimates to annual emissions estimates.
  • STPRFAC, flag value for use as a factor in variable TPFLAG in files PNTS and AREA to indicate that seasonal temporal-profile coefficients should be used for the indicated source.
  • MTPRFAC, flag value for use as a factor in variable TPFLAG in files PNTS and AREA to indicate that monthly temporal-profile coefficients should be used for the indicated source.
  • WTPRFAC, flag value for use as a factor in variable TPFLAG in files PNTS and AREA to indicate that day-of-week temporal-profile coefficients should be used for the indicated source.

Source code for the North Carolina UAM domain version of TMDIMS3.EXT
