QA Procedures For SMOKE Users

QA Internal to the SMOKE Programs

Data import programs in SMOKE follow a much more strict philosophy than do those in the UAM EPS -- the same philosophy that was decided on independently for EPS-95 (or GEMAP, as its earlier version was known. That philosophy is to report all the deficiencies in the input data to the user and to require that they be corrected before going further (in contrast to the EPS practice which attempts to continue processing with possibly-faulty records). The initial proof-of-concept model terminated processing with an error message upon the first error encountered; the production version is being reformulated to log all the errors that it can at a given stage of processing before terminating the programs. Errors diagnosed and reported during data import phase of SMOKE include the following:
  • badly formatted input files;
  • bad FIP codes, SCC codes, or variable names;
  • duplicate records;
  • missing entries for cross-references, profiles, etc.;
  • certain kinds of out-of-range data;
  • array-dimensioning errors and grid mis-matches; and
  • I/O errors.

Quality Assurance Program List

The following list in in alphabetical order:
  • CSGAREA: construct area source gridded speciated (optionally) controlled model emissions file AGTS
  • CSGMOBIL: Produce 1-layer gridded, speciated, controlled mobile source file MGTS
  • CSGPOINT: construct point source gridded, speciated, (optionally) hourly, (optionally) controlled emissions file PGTS or PGTSC
  • GRDAREA: Produce 1-layer gridded inventory species area source file AGRT or AGRD
  • GRDMOBIL:
  • GRDPOINT: construct point source gridded, (optionally) hourly, inventory pollutant emissions file PGRD

QA Data Set Definitions

  • AGRD and AGRT Area source (hourly time stepped or annual total) gridded inventory speciated emissions file
  • AGTS and AGTSC Area source hourly time stepped gridded speciated optionally-controlled model-emissions file
  • MGRD and MGRT MGRD and MGRT are the emissions prototype's mobile source gridded VMT files.
  • MGTS and MGTSC Mobile source hourly time stepped gridded speciated optionally-controlled model-emissions file
  • PGRD and PGRT Point source (hourly time stepped or annual total) gridded inventory speciated emissions file
  • PGTS and PGTSC Point source hourly time stepped gridded speciated optionally-controlled model-emissions file

AGRD and AGRT Files

Overview

Description: Sequence: Type: Origin: Used by: Summary:

File Format(s)

Same as: Structure: Header: Variables:

AGRD and AGRT are the emissions prototype's area source gridded inventory speciated files. They are portable, network transparent, machine architecture independent direct access binary files produced by program GRDAREA and used for analysis and quality assurance. AGRD is time-independent, with values representing inventory totals, and produced by using the inventory emissions file AREA as input to GRDAREA; AGRT is time-stepped, with values representing hourly emissions, and produced by using hourly time stepped emissions file ATMP as input to GRDAREA.

They are time-stepped or time-independent EDSS/Models-3 I/O API files of type GRDDED3 (gridded), with time step 1 hour, and dimensions appropriate for the model grid:

  • NLAYS, the number of layers, is 1
  • NCOLS and NROWS, the numbers of columns and rows, are defined in the GRDIMS3.EXT include file
The variables in the files are the gridded inventory speciated hourly time stepped area source emissions in units of tons per hour, or the gridded inventory speciated annual total area source emissions in units of tons per year. Names for the variables are the EINAM names from the CHDIMS3.EXT include file. For the North Carolina domain prototype, the list of variable names is the following:
  • CO for AIRS species 42101,
  • NOX for AIRS species 42603, and
  • VOC for AIRS species 43104.

AGTS and AGTSC Files

Overview

Description: Sequence: Type: Origin: Used by: Summary:

File Format(s)

Same as: Structure: Header: Variables:

AGTS is the emissions prototype's model-ready hourly time stepped gridded, optionally-controlled area source file. It is an (portable, network transparent, machine architecture independent direct access binary) file produced by program CSGAREA and used by the chemistry-transport model.

It is a time-stepped EDSS/Models-3 I/O API file of type GRDDED3 (gridded), with time step 1 hour, and dimensions appropriate for the model grid: the number of layers is 1, while the numbers NCOLS of columns and NROWS of rows are defined in the GRDIMS3.EXT include file. The variables in the file are the gridded model speciated hourly optionally-controlled time stepped area source emissions, in units of moles per hour. Names for the variables are the EMNAM names from the CHDIMS3.EXT include file. For the North Carolina domain UAM Carbon-Bond IV prototype, the list of variable names is the following:

  • CO, carbon monoxide;
  • NO, nitric oxide;
  • NO2, nitrogen dioxide;
  • OLE, olefins;
  • PAR, parafins;
  • TOL, toluene;
  • XYL, xylene;
  • FORM, formaldehyde;
  • ALD2, high molecular weight aldehydes;
  • ETH, ethene;
  • MEOH, methanol;
  • ETOH, ethanol; and
  • ISOP, isoprene

MGRD and MGRT Files

Overview

Description: Sequence: Type: Origin: Used by: Summary:

File Format(s)

Same as: Structure: Header: Variables:

MGRD and MGRT are the emissions prototype's mobile source gridded VMT files. They are portable, network transparent, machine architecture independent direct access binary files produced by program GRDMOBIL and used for analysis and quality assurance. MGRD is time-independent, with values representing inventory VMT totals, and produced by using the inventory emissions file MOBL as input to GRDMOBIL; MGRT is time-stepped, with values representing hourly vehicle type specific VMT, and produced by using hourly time stepped VMT file MTMP as input to GRDMOBIL.

They are time-stepped or time-independent EDSS/Models-3 I/O API files of type GRDDED3 (gridded), with time step 1 hour, and dimensions appropriate for the model grid:

  • NLAYS, the number of layers, is 1
  • NCOLS and NROWS, the numbers of columns and rows, are defined in the GRDIMS3.EXT include file
The variables in the files are the gridded inventory speciated hourly time stepped mobile source VMT in units of miles per hour, or the gridded inventory speciated annual total mobile source VMT in units of miles per year. The names for the variable in the file is VMT; in the case of MGRT, the file maps vehicle type into layer in the same fashion as in file MTMP.
